# Semantic Search Engine

**Due:** Monday by 11:59pm  
**Points:** 10  
**Submitting:** A text entry box or a file upload

---

## Objective
Build a production-ready semantic search application for a domain of your choice using embeddings and basic LLM integration.

---

## Requirements

### 1. Domain Selection (Alex)
*Get instructor approval.* Options include: 
* Academic papers, products, movies, news, reviews, legal docs, medical literature, recipes, jobs, or real estate.

### 2. Data Collection (Alex)
* Minimum of **100 documents**.
* Clean and preprocess the data.

### 3. Embedding Implementation (Syeda)
* **Model Choice:** Sentence Transformers, OpenAI embeddings, or Word2Vec.
* Create embeddings for the corpus.
* Implement **cosine similarity search**.
* Return **top-K** results.

### 4. LLM Enhancement (Choose ONE) (Shweta)
* **Option A:** Summarize top-K results.
* **Option B:** Answer questions based on results.
* **Option C:** Comparative analysis of results.

### 5. Interface (Neha)
* Deploy using the **command line**.
* **Bonus:** Deploy using Streamlit, Gradio, or HF Space on a publicly accessible URL. 

### 6. Code Quality (Fatima)
* Modular, documented code.
* Robust error handling.
* Inclusion of a `requirements.txt` file.
